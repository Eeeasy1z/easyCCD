# CCD上位机 MVP执行规划（冻结版）

## 1. 文档目的

本文件用于冻结当前阶段（MVP）的实现范围、技术口径与验收标准，作为开发与验收的唯一执行基线。

---

## 2. MVP范围定义

本阶段仅覆盖需求文档第9.1节“第一阶段：基础框架”目标：

1. 串口连接并接收CCD数据（128点灰度值）
2. 原始数据可视化显示（波形 + 数值）
3. 批量录制并导出CSV（用于离线查看与现场还原）

本阶段**不纳入验收**：
- 滤波模块
- 边缘检测模块
- 二值化模块
- 形态学处理
- 串口助手
- 二进制保存

---

## 3. 技术栈冻结

| 类别 | 选型 | 说明 |
|---|---|---|
| 编程语言 | Python 3.8+ | 开发效率高，便于快速迭代 |
| GUI框架 | PySide6 | Qt for Python，作为界面框架 |
| 波形绘制 | PyQtGraph | 实时绘图性能优于matplotlib |
| 串口通信 | pyserial | 串口收发与端口扫描 |
| 数值处理 | numpy | 基础数组处理 |

---

## 4. 串口参数口径（MVP）

1. 用户可配置项：**仅波特率**
2. 波特率范围：9600 / 19200 / 38400 / 57600 / 115200
3. 默认波特率：**115200**
4. 固定参数：8位数据位、无校验、1位停止位
5. 本阶段不支持：
   - 流控配置
   - 参数持久化（每次启动恢复默认）

---

## 5. 数据保存口径（MVP）

本阶段只提供：

- **批量录制并导出CSV**
- 支持提前设置CSV保存地址，并可重复复用
- 录制按钮为单按钮切换（开始录制/结束录制）
- 未设置保存地址时，点击开始录制弹窗要求先选择路径

本阶段不提供：

- 二进制保存（.bin）
- 扩展元数据导出

### 5.1 A方案增强（稳定优先）

在不扩展滤波/边缘检测等模块的前提下，增加稳定性能力：

1. 录制容量保护：支持最大缓存帧数（可选最大录制时长），超限时按策略处理（默认覆盖最旧帧）
2. 中断保护：串口断开或窗口关闭时，如有未导出录制数据，提示是否导出
3. 完整性统计：显示接收帧数、校验失败帧数、已录制帧数
4. 导出格式版本：CSV增加schema_version字段
5. 路径策略：支持默认录制目录 + 自动命名规则

## 6. 功能清单与交付项

### F1 串口通信
- 串口扫描
- 连接/断开
- 帧协议解析（0xAA 0x55 + 长度128 + payload + checksum）
- 异常处理（断开/读异常）

### F2 数据显示
- 波形显示：128点灰度曲线（X:0-127, Y:0-255）
- 数值显示：128点表格 + 最大/最小值及位置
- 状态栏：连接状态、帧号

### F3 数据保存
- 批量录制多帧数据并导出CSV（含帧号、时间戳、像素0~127）
- 单按钮切换录制状态（开始录制变红，二次点击结束并自动导出）
- 默认录制目录与自动命名
- 录制容量保护（最大缓存帧数）
- 中断导出保护（断开/关闭前提示）
- 录制完整性统计（接收/坏帧/录制帧）

### F4 程序入口与主界面
- MainWindow主界面布局
- 程序入口可启动并正常关闭

## 8. MVP验收标准（冻结）

通过标准：

1. 能扫描并列出可用串口
2. 能打开串口并接收有效帧数据
3. 波特率可由用户修改，默认115200
4. 不要求流控与参数持久化
5. 波形控件可实时显示128点灰度曲线
6. 数值控件可显示128像素值及最大/最小值位置
7. 可执行批量录制并导出CSV
8. 支持提前设置并复用CSV保存路径；未设置路径时点击开始录制可弹窗引导设置
9. 录制按钮可在“开始录制/结束录制”之间切换，结束后自动保存
10. 本阶段不要求二进制保存
11. 支持最大缓存帧数并在超限时按预设策略处理
12. 串口断开或程序关闭时，若有未导出录制数据可提示导出
13. 界面可显示接收帧数、校验失败帧数、已录制帧数
14. 导出CSV包含schema_version字段
15. 界面交互流畅，无明显卡顿

不通过示例：
- 只能显示数据但无法批量录制导出CSV
- 串口连接成功但无法解析128点有效帧
- 默认波特率不是115200且无明确设置
