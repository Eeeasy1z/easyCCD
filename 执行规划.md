# CCD上位机执行规划（最小可执行方案）

## 1. 方案概述

本规划基于需求文档第9.1节"第一阶段：基础框架"，目标是搭建一个最小可运行版本（MVP），实现核心的串口通信和数据显示功能。

### 1.1 核心目标

- [x] 串口连接并接收CCD数据（128像素，0-255灰度值）
- [x] 波形显示128点数据
- [x] 数值显示128点数据
- [x] 数据保存功能

### 1.2 技术选型

| 组件 | 技术选型 | 理由 |
|------|----------|------|
| 编程语言 | Python 3.8+ | 跨平台，生态丰富，便于快速开发 |
| GUI框架 | PySide6 | 官方Qt for Python绑定，许可友好，支持复杂界面 |
| 串口通信 | pyserial | Python标准串口库，稳定可靠 |
| 波形绘制 | PyQtGraph | 比matplotlib更快，适合实时显示 |
| 数据存储 | JSON/CSV | 简单易用，便于后续处理 |

## 5. 串口通信协议

根据需求文档第5节：

| 字段 | 长度 | 说明 |
|------|------|------|
| 帧头 | 2字节 | 0xAA 0x55 |
| 数据长度 | 1字节 | 128 |
| 数据内容 | 128字节 | 128个像素灰度值 |
| 校验码 | 1字节 | 累加和模256 |

## 8. 验收标准

完成第一阶段后，软件应满足：

- [ ] 能够扫描并列出可用串口
- [ ] 能够打开串口并接收数据
- [ ] 波特率可由用户修改，默认值为115200
- [ ] 本阶段不要求流控配置和参数持久化
- [ ] 波形控件实时显示128点灰度曲线
- [ ] 数值控件显示128个像素值及最大/最小值
- [ ] 能够保存单帧数据为CSV
- [ ] 本阶段不要求二进制保存与连续保存
- [ ] 界面响应流畅，无明显卡顿
