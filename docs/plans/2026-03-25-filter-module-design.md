# 滤波模块设计方案（V1，非需求主文档）

> 说明：本文件是历史设计方案（implementation/design note），**不是需求主文档**。  
> 需求唯一入口：`D:\GitHub_Repository\easyCCD\需求文档.md`。

## 1. 目标与边界

### 1.1 本次目标（V1）
- 在现有上位机中增加可配置的一维滤波处理链
- 用户可点击添加滤波器、调整顺序、修改参数并实时观察效果
- 首批实现3个核心滤波器：
  - 一维均值滤波
  - 一维中值滤波
  - 一维高斯滤波

### 1.2 明确不做（V1排除项）
- 双边滤波（已在需求中排除）
- 形态学操作（属于二值化阶段，不放入滤波模块）
- 一次性实现全部6种滤波

---

## 2. 设计输入（需求约束）

1. 输入数据是离散CCD一维数组，长度128，灰度值0~255
2. 滤波必须是数字式一维处理
3. 滤波链必须支持：
   - 点击添加
   - 顺序可调（上移/下移）
   - 参数实时生效
4. 结果必须实时展示，便于比较方案
5. 需保留后续扩展接口，支持新增滤波器

---

## 3. 方案对比

### 方案A：最小闭环直连主窗口（推荐）

**做法**
- 在主窗口中增加滤波链配置区
- 增加轻量处理层（Filter接口 + Pipeline）
- 在线数据与回放数据统一走一条滤波执行路径

**优点**
- 改动范围可控，落地快
- 风险低，适合分批验证节奏

**风险**
- 主窗口逻辑会增加，需要控制职责边界

### 方案B：先完整抽象处理框架再接UI

**做法**
- 先构建完整processing子系统，再接入UI

**优点**
- 架构更“理想化”

**风险**
- 首轮交付慢，回归成本高，不利于当前迭代节奏

### 方案C：只做UI框架占位

**做法**
- 只做配置界面，不接实际滤波计算

**优点**
- 界面开发快

**风险**
- 业务价值低，不能验证实际算法效果

**结论：采用方案A。**

## 4. 架构设计

### 4.1 模块结构

新增文件建议：

```
src/core/
  filter_pipeline.py
  filters/
    __init__.py
    base.py
    mean.py
    median.py
    gaussian.py
```

现有文件修改：
- `src/ui/main_window.py`：增加滤波配置UI与管线调用
- `src/ui/widgets/waveform_widget.py`：支持原始曲线与滤波后曲线对比显示

### 4.2 数据流

统一处理路径：
- 在线串口：`_on_data_received(raw)`
- 离线回放：`_show_playback_frame(raw)`

两者都调用：
`filtered = pipeline.apply(raw)`

展示策略（V1）：
- 波形图同时显示原始曲线 + 滤波后曲线
- 表格默认显示滤波后数据

## 5. 接口定义（V1）

### 5.1 Filter接口

- `name: str`
- `default_params: dict[str, float | int]`
- `apply(data: list[int], params: dict) -> list[int]`

### 5.2 Pipeline接口

- `set_steps(steps)`
- `add_step(filter_name, params)`
- `remove_step(index)`
- `move_up(index)` / `move_down(index)`
- `apply(data)`

---

## 6. 算法与参数规则

### 6.1 均值滤波
- 参数：窗口大小（奇数，默认5）

### 6.2 中值滤波
- 参数：窗口大小（奇数，默认5）

### 6.3 高斯滤波
- 参数：sigma（默认1.0）
- 卷积核根据sigma动态生成并归一化

### 6.4 通用边界规则
- 输入长度固定128，输出长度必须128
- 边界采用镜像填充
- 窗口参数非法时自动修正到最近合法值

---

## 7. UI交互设计（V1）

1. 左侧“可用滤波器”列表：均值/中值/高斯
2. 点击某个滤波器 => 添加到“当前滤波链”
3. 每个步骤支持：
   - 上移
   - 下移
   - 删除
   - 参数输入（spinbox/slider）
4. 参数或顺序变化后立即重算并刷新

---

## 8. 错误处理

- 参数越界：自动修正并在状态栏提示
- 滤波执行异常：回退到原始数据并提示错误
- 空滤波链：输出等于原始输入

---

## 9. 验收标准（V1）

1. 可添加3种滤波器并形成链式处理
2. 可调整滤波顺序并看到输出变化
3. 参数变化可实时生效
4. 在线数据与回放数据均走同一滤波链
5. 输出数据长度与范围合法（128点、0~255）
6. 界面操作流畅，无明显卡顿

---
